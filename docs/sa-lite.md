# SA-Lite

## 使用者角色

單一個人使用者(擁有者)。透過瀏覽器使用;未來加 LINE bot / PWA 入口。

## 使用情境

1. 看到有用資訊 → 貼進收件匣 → 系統歸檔
2. 想找之前存的東西 → 搜尋 / 篩選 → 開詳情
3. 分類錯了 → 手動改類型 / 屬性
4. 規劃某場景 → 把多筆 entry 組成 collection(Phase 4)

## 功能清單(Phase 1)

| 功能 | 輸入 | 輸出 |
|---|---|---|
| 新增 entry | 標題、類型、屬性、摘要、內容、來源、標籤 | 一筆 entry 寫入 DB |
| 編輯 / 刪除 entry | entry id + 欄位 | 更新 / 移除 |
| 卡牆瀏覽 | 領域 / 類型 / 標籤篩選 | entry 卡片清單 |
| 關鍵字搜尋 | 關鍵字 | 命中標題 / 摘要 / 內容的 entry |
| entry 詳情 | entry id | 完整內容頁 |

## 主要流程

新增 → 選類型 → 依 `attrs_schema` 動態長屬性欄位 → 填 → 存 → 卡牆即時更新。

## 例外流程

- 未設定 Supabase 連線 → 顯示提示、degrade,不崩潰
- 無法建立 session(未開 anonymous sign-in)→ 顯示錯誤訊息與修法
- 儲存 / 刪除失敗 → 顯示錯誤,不假裝成功

## 畫面草稿

- **收件匣頁**(`/`):標題列 + 新增鈕、篩選列(搜尋 / 領域 / 類型 / 標籤)、卡牆(RWD 1→2→3 欄)、新增 / 編輯 Modal

- **404**(`/:pathMatch`)
