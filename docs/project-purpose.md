# Project Purpose

## 背景

網路上發現的有用資訊(美食地圖、AI 技能、前端技巧、旅遊資訊等)散落在 LINE 收藏與各處,無法歸檔、無法找到、無法重複利用。

## 問題

- 存了等於沒存:找不回來
- 沒有結構:同一份資料無法服務多個場景
- 手動整理成本高

## 目標

一個「收件匣式」的個人知識網站:

- **一丟進去就好**:貼文字 / URL / 照片,系統自動判斷類型、歸檔
- **找得到**:關鍵字 + 語意搜尋
- **可調整**:手動改分類、待確認佇列、新增類型
- **模組化重用**:同一筆資料被多個場景(行程、地圖、儀表板)引用
- **知識辨識**:資料間的關聯可視覺化
- **可延伸**:未來長成美食工作站、城市地圖知識庫等大型分區 view

## 不做什麼

- 不做多人協作平台(個人單一使用者)
- 不做全自動、無人審核的分類(避免 taxonomy drift)
- MVP 不做登入 UI、graph view、LINE bot、PWA、本地 LLM

## MVP 範圍(Phase 1)

手動新增 / 編輯 / 刪除 entry(schema-driven)、卡牆瀏覽、關鍵字搜尋、標籤篩選。
**一天不接 LLM 也能用** = 驗證資料模型與流程順不順。

## 使用者角色

單一個人使用者(擁有者本人)。

## 核心流程

收內容 → (LLM)分類抽取 → 信心分流(直接歸檔 / 待確認)→ 卡牆瀏覽搜尋 → 需要時組合成 collection。
